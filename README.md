📝 Form Design & Validation

A secure and accessible form validation project built with HTML5, CSS3, and Vanilla JavaScript, focusing on user input validation, real-time feedback, accessibility standards, and data integrity using the IPO (Input → Process → Output) pattern.

📖 Overview

Form Design & Validation is a frontend development project completed during my Frontend Development Internship at Decodelabs. The project focuses on designing a user-friendly and accessible form while implementing robust client-side validation techniques to ensure accurate and secure user input.

✨ Features

- Semantic Form Structure
- Real-Time Form Validation
- Email Validation using Regular Expressions
- Strong Password Validation
- Dynamic Error and Success Messages
- Custom Form Submission Handling
- Accessible Form Design
- ARIA Attribute Integration
- Responsive User Interface
- Secure DOM Updates

🛠️ Technologies Used

- HTML5
- CSS3
- Vanilla JavaScript
- Regular Expressions (Regex)
- ARIA Accessibility Attributes

🧠 IPO Pattern

Input

- User enters form data.
- User submits the form.

Process

- JavaScript intercepts the default form submission.
- Validation logic checks email format and password strength.
- Error and success states are generated based on input validity.
- Accessibility attributes are updated dynamically.

Output

- Real-time validation feedback.
- Error messages for invalid inputs.
- Success messages for valid submissions.
- Improved accessibility and user experience.

🔍 Validation Features

Email Validation

- Validates proper email structure.
- Prevents invalid email formats.
- Uses Regular Expressions (Regex) for pattern matching.

Password Validation

Ensures passwords contain:

- Minimum required length
- Uppercase letters
- Lowercase letters
- Numeric characters
- Special characters

♿ Accessibility Features

- Semantic HTML Form Elements
- Proper Label-to-Input Associations
- "aria-invalid" for validation state indication
- "aria-describedby" for linking error messages
- "aria-live="polite"" for screen reader announcements
- Improved keyboard and screen reader support

🔒 Security & Best Practices

- No use of "innerHTML"
- Safe content updates using "textContent"
- Client-side validation before submission
- Clean separation of structure, style, and logic
- Maintainable and modular JavaScript functions
